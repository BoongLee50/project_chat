> 최종 업데이트: 2026-07 (정적 UI 완료 · 채팅 보관정책 확정).
> 이 문서는 "어디까지 했고 / 무엇이 남았고 / 무엇을 정해야 하는지"의 핸드오프용 요약이다.
> 설계 상세는 [01 프로토콜/API](01-protocol-api-spec.md) · [02 DB 스키마](02-db-schema.md) · [03 Flutter 구조](03-flutter-structure.md) 참고.

## 1. 완료한 작업

**환경 · 저장소**
- Git 초기화 + GitHub 연결(소스트리 워크플로우) + `.gitignore` 점검
- 멀티기기(랩탑/Mac) 개발 환경 세팅 가이드
- 릴리즈 APK 빌드 (테스트 서명, 약 47MB) — `build/app/outputs/flutter-apk/app-release.apk`

**설계 문서** (`docs/`)
- 01 통신 프로토콜 & REST/WebSocket API 명세
- 02 DB 스키마 (PostgreSQL + Redis)
- 03 Flutter 폴더 구조 설계
- 시스템 아키텍처 · 화면 흐름 다이어그램

**앱 UI — 화면 11개** (정적, 다크 테마 고정, Android 에뮬레이터 검증 완료)
- 로그인 → 온보딩 3단계(닉네임 · 출생년도 · 성별/국가)
- 메인 셸(하단 5탭 IndexedStack):
  - 포스트(오늘의 포스트/홈) · 달빛가든(피드) · 대화방 → 채팅창 · 친구 · 프로필
- 디자인 시스템(`app_colors`/`app_dimens`/`app_theme`), feature-first 구조, 공용 하단 내비(`MainBottomNav`)

> 현재는 "정적 UI" = 코드로 만든 화면 + 더미 데이터/사진. 상호작용(입력·검증·탭 전환·화면 이동)은 동작하나, 서버 데이터·실기능은 미연결.

## 참고 — 재개 방법
1. 다른 기기: `git clone` → `flutter pub get` → (Android) `flutter run -d <device>`
2. 앱 첫 화면은 로그인. 온보딩 완료 시 메인 셸(5탭) 진입.
3. 검증: Android 에뮬레이터 + `adb exec-out screencap`, 커밋 전 `flutter analyze` + `flutter test`.
